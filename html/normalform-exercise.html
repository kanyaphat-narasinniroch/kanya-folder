TASKS — สิ่งที่ต้องทำ
  • 1 ระบุว่า Primary Key ของตารางนี้คืออะไร และตารางนี้ผ่าน 1NF แล้วหรือไม่
  • 2 จาก FD ด้านบน ระบุว่า FD ใดคือ Partial Dependency (ขึ้นกับแค่บางส่วนของ PK)
  • 3 แยกตารางเพื่อกำจัด Partial Dependency ให้ผ่าน 2NF (จะได้กี่ตาราง?)
  • 4 ระบุ Primary Key และ Foreign Key ของทุกตารางที่ได้

คำใบ้:
• Primary Key ของตารางนี้คือ รหัสการยืม เพียงตัวเดียว
• เนื่องจาก PK เป็น attribute เดียว ปัญหาจึงไม่ใช่ Partial Dependency แบบคลาสสิก
• แต่ข้อมูลสมาชิกและหนังสือยังซ้ำซ้อนอยู่ — ควรแยกออกไปเป็นตารางแยก
• เป้าหมาย: แยกเป็น 3 ตาราง → สมาชิก, หนังสือ, การยืม
• FD: รหัสสมาชิก → ชื่อ, เบอร์ และ รหัสหนังสือ → ชื่อ, สนพ., ราคา คือ FD ที่ควรแยก

  • 4 ในตารางที่ได้จาก 2NF มี Transitive Dependency อะไรบ้าง?
  • 5 แยกตารางให้ผ่าน 3NF ตารางสุดท้ายควรมีกี่ตาราง? อะไรบ้าง?

• PK คือ (รหัสนักศึกษา + รหัสวิชา) — Composite Key
• หลังแยก 2NF จะได้ 3 กลุ่ม: นักศึกษา, วิชา, การลงทะเบียน
• Transitive Dependency ซ่อนอยู่ใน 2 ที่: นักศึกษา → สาขา → คณะ และ วิชา → อาจารย์ → ชื่ออาจารย์
• ตารางสุดท้ายควรได้ 5 ตาราง: นักศึกษา, สาขา, วิชา, อาจารย์, การลงทะเบียน

TASKS — สิ่งที่ต้องทำ
  • 1 ระบุ FD ทั้งหมด ที่พบในตาราง รวมถึง FD ที่ซ่อนอยู่ในข้อมูล (ดูจากข้อมูลจริงในตาราง)
  • 2 แยกตารางให้ผ่าน 3NF ก่อน ระบุจำนวนและชื่อตารางที่ได้
  • 3 ตรวจสอบทุก FD ในแต่ละตาราง: มี FD ใดที่ตัวกำหนด (Determinant) ไม่ใช่ Superkey บ้าง?

  • 5 ระบุจำนวนตารางสุดท้าย พร้อม Primary Key และ Foreign Key ของทุกตาราง
  • 6 Bonus: การแยกตาราง BCNF ในข้อนี้ทำให้ข้อมูลสูญหาย (Lossy) หรือไม่? อธิบาย
